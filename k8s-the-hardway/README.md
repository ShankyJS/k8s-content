## Why do we need a CA and TLS Certificates?

Certificate Authority (CA) and we can use the CA to generate several certificates.

Certificates are used to confirm (authenticate) identity. THey are used to prove that youa re who you say you are.

A ***A certificate authority*** provides the ability to confirm that a certificate is valid. A certificate authority can be used to validate any certificate that was issued using that CA.

Kubernetes uses certificates for a avariety of security functions, and the different parts of our cluster will validate certificates using the certificate authority.

#### What Certificates Do We need?

We need:

***Client certificates***: These certificates provide client authentication for various users: admin, kube-controller-manager, kube-proxy, kube-scheduler, and the kubelet client on each worker node.

***Kubernetes API Server Certificate***: THis is the TLS certificate for the K8s API

***Service Account Key Pair:*** Kubernetes uses a certificate to sign service account tokens, so we need to provide a certificate for that purpose.

### Provisioning a Certificate Authority (CA)

Here we need cfssljson and cfssl so we can install it like this:

````
sudo curl -s -L -o /bin/cfssl https://pkg.cfssl.org/R1.2/cfssl_linux-amd64
sudo curl -s -L -o /bin/cfssljson https://pkg.cfssl.org/R1.2/cfssljson_linux-amd64
sudo curl -s -L -o /bin/cfssl-certinfo https://pkg.cfssl.org/R1.2/cfssl-certinfo_linux-amd64
sudo chmod +x /bin/cfssl*
````


Now we can use this command to generate the certificate authority.

````
{

cat > ca-config.json << EOF
{
  "signing": {
    "default": {
      "expiry": "8760h"
    },
    "profiles": {
      "kubernetes": {
        "usages": ["signing", "key encipherment", "server auth", "client auth"],
        "expiry": "8760h"
      }
    }
  }
}
EOF

cat > ca-csr.json << EOF
{
  "CN": "Kubernetes",
  "key": {
    "algo": "rsa",
    "size": 2048
  },
  "names": [
    {
      "C": "US",
      "L": "Portland",
      "O": "Kubernetes",
      "OU": "CA",
      "ST": "Oregon"
    }
  ]
}
EOF

cfssl gencert -initca ca-csr.json | cfssljson -bare ca

}
````

The output will generate several files. 

ca-key.pem (Private certificate for CA)
ca.pem (Public certificate, this is used on several parts of K8s)

## Moving files to the cloud servers that has our K8s clusters.

Now that certificates have been generated, we need to move the files onto the appropiate servers. 

````
Move certificate files to the worker nodes:

scp ca.pem <worker 1 hostname>-key.pem <worker 1 hostname>.pem user@<worker 1 public IP>:~/
scp ca.pem <worker 2 hostname>-key.pem <worker 2 hostname>.pem user@<worker 2 public IP>:~/
Move certificate files to the controller nodes:

scp ca.pem ca-key.pem kubernetes-key.pem kubernetes.pem \
    service-account-key.pem service-account.pem user@<controller 1 public IP>:~/
scp ca.pem ca-key.pem kubernetes-key.pem kubernetes.pem \
    service-account-key.pem service-account.pem user@<controller 2 public IP>:~/
````
